1. Introduzione

Amar SAF (Security & Admin Framework) è un plugin WordPress modulare che trasforma un qualsiasi tema in un sito professionale con strumenti di sicurezza, SEO, performance, amministrazione e documentazione.

Creato da Amar Amoretti per sviluppatori e agenzie che vogliono un backend brandizzato e funzionale, senza dipendenze pesanti.

Filosofia: Ogni modulo fa una cosa e la fa bene. Zero jQuery, zero bloat, zero dipendenze esterne.

Cosa include

  • 9 moduli funzionali indipendenti
  • 10 tab in ⚙️ Dati Sito per configurare tutto
  • 7 tab in 📖 Guida Sito per documentazione
  • Child theme agnostico (funziona con qualsiasi tema)
  • Internazionalizzazione (Italiano / Inglese)

2. Installazione

2.1 Come MU Plugin (consigliato)

  1. Carica la cartella saf/ in /wp-content/mu-plugins/saf/
  2. Carica saf-loader.php in /wp-content/mu-plugins/saf-loader.php
  3. I moduli sono subito attivi — non serve attivarli
Vantaggio: il plugin non può essere disattivato accidentalmente.

2.2 Come Plugin Normale

  1. Comprimi il contenuto della cartella saf/ in un file .zip
  2. Vai su Plugin → Aggiungi → Carica Plugin
  3. Carica lo zip e attiva
Il plugin si chiama Amar SAF nell'elenco plugin.

3. ⚙️ Dati Sito — I Tab

La pagina principale di configurazione. Ogni tab gestisce un'area specifica.

3.1 🏢 Organizzazione

Dati principali del sito/azienda:

  • Nome organizzazione — usato in JSON-LD, email, shortcode
  • URL sito web — default: homepage
  • Logo — seleziona con media uploader
  • Indirizzo, CAP, Città, Partita IVA — per NAP e fatturazione
  • Email, Telefono — contatti principali
  • Social — Facebook, Instagram, YouTube, LinkedIn, Twitter

Questi dati alimentano lo shortcode [saf_footer_info] e il JSON-LD Organization in homepage.

3.2 🔍 SEO & Immagini

Immagini di fallback per Open Graph:

  • Immagine OG Default — usata quando una pagina non ha featured image
  • Immagine OG Secondaria — alternativa

Le immagini devono essere almeno 1200×630px per funzionare bene su social.

3.3 🔒 Sicurezza

  • Tentativi login massimi (3-20) — dopo il limite, IP bloccato 15 minuti
  • Nascondi slug autore — /author/nome va in 404 (anti-enumerazione)
  • Abilita sitemap XML — genera /sitemap.xml semplice

3.4 🤖 Robots.txt

Editor per il contenuto di robots.txt. La sitemap è auto-compilata. Utile per personalizzare le regole di crawling.

3.5 📍 NAP Footer

HTML personalizzato per il footer NAP (Nome, Indirizzo, Telefono). Richiamabile con [saf_nap_html].

3.6 ⚙️ Avanzate

  • Email — Mittente: nome e indirizzo per tutte le email WordPress
  • Commenti: disabilita commenti su tutto il sito
  • HSTS: Strict-Transport-Security (solo con SSL stabile)
  • Upload SVG: abilita upload di file SVG con sanitizzazione automatica (rimozione script e event handler)
  • Nascondi tab Progetto: utile dopo la consegna al cliente
  • Menu Admin: nascondi voci a editor/autori (Strumenti, Commenti, Plugin, ecc.)
  • Disinstallazione: pulisci tutte le opzioni SAF dal database

3.7 🎨 Child Theme

  • Auto-create: crea il child theme amar-design con un clic dalla dashboard o dal tab, parametri configurabili
  • Rilevamento intelligente: se un child theme diverso da amar-design è attivo, mostra i dettagli senza permettere reset
  • Editor style.css: modifica contenuto da backend con salvataggio immediato
  • Visualizzatore functions.php: textarea read-only per ispezione rapida
  • Diagnostica: percorsi, permessi, proprietario file, metodo scrittura (WP_Filesystem o diretto)
  • Notifiche: avviso OPcache se modifiche non subito visibili, avviso attivazione child theme, notice auto-create
  • Header Search Divi: attivazione overlay AJAX fullscreen (se tema parent Divi)

3.8 📝 Crediti

  • Nome autore e sito web
  • Nome cliente e sito web
  • Note di sviluppo (visibili solo nel backend)
  • Data creazione

3.9 🛠 Plugin e Strumenti

Tab dedicato al rilevamento e checklist dei plugin e strumenti del sito:

  • .htaccess — visualizzazione read-only del file .htaccess (se presente)
  • Cache Lato Server — rilevamento OPcache, Memcache, Redis, APCu, Object Cache persistente, Reverse Proxy/Varnish, costante WP_CACHE
  • Cache Lato Client/Tema — rilevamento Divi et-cache (conteggio file se Divi attivo), 12 plugin cache (W3 Total Cache, WP Super Cache, WP Rocket, Flying Press, ecc.), checkbox di conferma
  • SEO Plugin — rilevamento di 11 plugin SEO (Yoast, Rank Math, SEOPress, AIOSEO, ecc.)
  • Sitemap — rilevamento plugin sitemap (Google XML Sitemaps, sitemap built-in Yoast/Rank Math)
  • Google Analytics & Search Console — rilevamento plugin analytics (Site Kit, MonsterInsights, ExactMetrics, ecc.)
  • Backup — rilevamento plugin backup (UpdraftPlus, BackWPup, Duplicator, Jetpack, ecc.) + checklist hosting/off-site
  • Sicurezza — rilevamento 10 plugin sicurezza (Wordfence, Solid Security, Sucuri, All In One WP Security, WPScan, Cerber, Defender, Anti-Malware, SecuPress, Jetpack Protect) + checklist 5 voci (firewall, protezione login, scan malware, SSL, aggiornamenti automatici)
  • Checkbox di conferma per ogni sezione: spunta ciò che è configurato e funzionante

3.10 ⚙️ Sistema

Tab informativo con diagnostica server, opzioni SAF e self-test:

  • Info Server — PHP version, WordPress version, tema attivo, permalink structure, memory limit, max execution time, upload max
  • Costanti SAF — elenco di tutte le costanti definite da SAF con relativo valore
  • File SAF — verifica esistenza di tutti i file del plugin (admin.php, security.php, seo.php, ecc.)
  • Language Files — verifica caricamento lingue it_IT e en_US e conteggio chiavi tradotte
  • Options SAF — elenco opzioni salvate nel database con valori
  • Self-Test completo — pulsante per eseguire diagnostica automatica che verifica: costanti SAF obbligatorie (12), file plugin (10), language files (2), chiavi traduzione (12 obbligatorie), opzioni DB (3+), funzione saf_write_file(), costanti WordPress (WP_DEBUG, WP_DEBUG_DISPLAY, DISALLOW_FILE_EDIT, WP_CACHE, WP_MEMORY_LIMIT)
  • Risultati con semaforo: ✅ OK, ⚠️ WARN, ❌ ERR

4. Moduli SAF

4.1 🔒 Sicurezza (security.php)

  • Rate limiting login — blocca IP dopo N tentativi falliti
  • XML-RPC disabilitato — prevenzione attacchi brute force
  • HSTS — header Strict-Transport-Security (opzionale, da attivare)
  • Honeypot — campo nascosto anti-bot nel login
  • Login brandizzato — logo e stile personalizzati
  • Security headers — X-Content-Type-Options, X-Frame-Options
  • Disabilita author enum — /author/slug → 404

4.2 ⚙️ Admin (admin.php)

Il modulo più grande. Gestisce:

  • Pagina ⚙️ Dati Sito con 10 tab (Org, SEO, Security, Robots, NAP, Child, Credits, Adv, Plugin, Sistema)
  • Registrazione settings e sanitizzatori
  • Helper saf_get_org_data() — array dati organizzazione
  • Media uploader per logo
  • Menu admin brandizzato arancione
  • SMTP from name/email
  • Pulizia widget dashboard
  • Disinstallazione opzioni DB

4.3 🔍 SEO (seo.php)

  • JSON-LD Organization — homepage
  • JSON-LD WebSite — homepage
  • BreadcrumbList — breadcrumb JSON-LD
  • Canonical URL — per tutte le pagine
  • Open Graph tags — og:title, og:description, og:image, og:url
  • Meta description fallback

Non sostituisce Rank Math / Yoast: questi tag si attivano solo se Rank Math è assente.

4.4 🛠️ Helpers (helpers.php)

  • saf_youtube_id() — estrae ID YouTube da URL
  • saf_date_ita() — data in formato italiano
  • saf_social_share_link() — link condivisione social
  • saf_pagination() — paginazione WP
  • saf_reading_time() — tempo di lettura shortcode
  • saf_nap_html() — shortcode NAP footer
  • saf_footer_info() — shortcode info footer
  • saf_truncate() — tronca testo

4.5 🚀 Performance (performance.php)

  • oEmbed disabilitato — blocca embedding esterni non necessari
  • Heartbeat control — riduce frequenza heartbeat (60s)
  • Revisioni limitate — massimo 5 revisioni per post
  • Lazy load — immagini e iframe nativi
  • CF7 ottimizzato — carica JS/CSS solo dove serve
  • jQuery migrate off — se non usi plugin che lo richiedono
  • Cache preconnect — hint per Google Analytics, CDN

4.6 🧹 Cleanup (cleanup.php)

  • Commenti disabilitati — se attivato da Avanzate
  • Menu admin nascosto — se configurato
  • Admin bar — menu personalizzato
  • Footer admin — brandizzato con crediti sviluppatore
  • Update footer — rimosso (versione WP)

4.7 📊 Dashboard (dashboard.php)

  • Widget Panoramica Sito in homepage admin
  • Avanzamento checklist pre-go-live con barra colorata
  • Info sito, tema, organizzazione
  • Pulsanti rapidi per Dati Sito e Guida
  • Crediti sviluppatore (da tab Crediti)
  • Sezione sicurezza (stato SSL, XML-RPC, HSTS, plugin)

4.8 📖 Guida (guida.php)

Pagina di documentazione integrata con 7 tab:

  • Info — crediti, licenza, disclaimer, guide
  • Struttura — mappa file del tema
  • Shortcode — reference copia-incolla
  • Sicurezza — dettaglio misure attive
  • Checklist — pre-go-live con stato salvato
  • Progetto — documenti PDF con date e note
  • Note Dev — appunti sviluppatore

4.9 🖼 Upload SVG

Modulo integrato in saf-loader.php per la gestione sicura di file SVG:

  • Abilitazione da ⚙️ Dati Sito → Avanzate → Upload SVG
  • Sanitizzazione DOM-based: whitelist di tag e attributi SVG sicuri
  • Rimozione automatica di script, event handler (onclick, onload, onerror...) e javascript: href
  • Supporto MIME corretto per file .svg e .svgz
  • Filtri WordPress: upload_mimes, wp_check_filetype_and_ext, wp_handle_upload_prefilter
⚠️ Per sicurezza massima in produzione, installa la libreria enshrined/svg-sanitize via Composer.

5. Child Theme

Il child theme amar-design è un tema figlio preconfigurato che SAF mette a disposizione. Non viene creato automaticamente — SAF chiede all'utente se desidera crearlo tramite un avviso nella dashboard.

Perché serve un child theme? Se modifichi i file del tema principale (Divi, Astra, Twenty Twenty-Four…), le tue personalizzazioni vengono sovrascritte a ogni aggiornamento del tema. Un child theme mantiene le modifiche al sicuro.

5.1 Creazione

  1. Vai su ⚙️ Dati Sito → Child Theme
  2. Se non esiste, clicca "Crea child theme amar-design"
  3. SAF copia style.css, screenshot.png, functions.php e le cartelle inc/, css/, js/
  4. Modifica il campo Template con la cartella del tuo tema (es. Divi, astra, twentytwentyfour)
  5. Attiva il tema da Aspetto → Temi

Se il child theme esiste già, puoi usare il pulsante "Resetta e ricrea" con doppia conferma per cancellare tutto e ricreare da zero.

5.2 style.css

Il file style.css del child theme ha una riga obbligatoria:

Template: NOME-CARTELLA-TEMA-PRINCIPALE

Sostituisci YOUR-THEME con il nome esatto della cartella del tuo tema (es. Divi, astra, twentytwentyfour).

Puoi modificare style.css dal backend: ⚙️ Dati Sito → Child Theme.

5.3 Header Search Divi

Attivabile da ⚙️ Dati Sito → Child Theme → Funzionalità Divi.

  • Aggiunge icona lente nel menu mobile
  • Overlay fullscreen per la ricerca
  • Ricerca AJAX in tempo reale su titoli
  • Scorciatoia da tastiera: Ctrl+K o ⌘K

Funziona solo se il tema parent è Divi.

8. Disinstallazione

  1. Disattiva il plugin o rimuovi saf-loader.php (se MU)
  2. Vai in ⚙️ Dati Sito → Avanzate → Pulisci tutte le opzioni SAF
  3. Oppure esegui SQL: DELETE FROM wp_options WHERE option_name LIKE 'saf_%';
Nota: Le opzioni salvate rimangono nel database anche a plugin disattivato. Usa la pulizia manuale per rimuoverle.
