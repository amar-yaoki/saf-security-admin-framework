1. Introduction

Amar SAF (Security & Admin Framework) is a modular WordPress plugin that turns any theme into a professional website with security, SEO, performance, administration, and documentation tools.

Created by Amar Amoretti for developers and agencies who want a branded, functional backend without heavy dependencies.

Philosophy: Each module does one thing and does it well. Zero jQuery, zero bloat, zero external dependencies.

What's included

  • 9 functional modules — independent and focused
  • 10 tabs in ⚙️ Site Data for full configuration
  • 7 tabs in 📖 Site Guide for documentation
  • Generic child theme — works with any parent theme
  • Internationalization (English / Italian)

2. Installation

2.1 As MU Plugin (recommended)

  1. Upload the saf/ folder to /wp-content/mu-plugins/saf/
  2. Upload saf-loader.php to /wp-content/mu-plugins/saf-loader.php
  3. Modules are active immediately
Benefit: the plugin cannot be accidentally deactivated.

2.2 As Regular Plugin

  1. Compress the contents of the saf/ folder into a .zip file
  2. Go to Plugins → Add New → Upload Plugin
  3. Upload the zip and activate
The plugin is listed as Amar SAF in your plugin list.

3. ⚙️ Site Data — The Tabs

3.1 🏢 Organization

Main site/company data:

  • Organization name — used in JSON-LD, emails, shortcodes
  • Website URL — defaults to homepage
  • Logo — select via media uploader
  • Address, ZIP, City, VAT — for NAP and invoicing
  • Email, Phone — main contacts
  • Social — Facebook, Instagram, YouTube, LinkedIn, Twitter

3.2 🔍 SEO & Images

Fallback images for Open Graph:

  • Default OG Image — used when a page has no featured image
  • Secondary OG Image — alternative

3.3 🔒 Security

  • Max login attempts (3-20) — IP blocked for 15 minutes after limit
  • Hide author slugs — /author/name returns 404 (prevents user enumeration)
  • Enable XML sitemap — generates simple /sitemap.xml

3.4 🤖 Robots.txt

Editor for robots.txt content. The sitemap URL is auto-filled.

3.5 📍 NAP Footer

Custom HTML for the NAP footer (Name, Address, Phone). Use [saf_nap_html] shortcode.

3.6 ⚙️ Advanced

  • Email — Sender: name and address for all WordPress emails
  • Comments: disable sitewide
  • HSTS: Strict-Transport-Security header (stable SSL only)
  • SVG Upload: enable SVG file upload with automatic sanitization
  • Hide Project tab: useful after client delivery
  • Admin Menu: hide items from editors/authors
  • Uninstall: delete all SAF options from database

3.7 🎨 Child Theme

  • Auto-create: create the amar-design child theme with one click from dashboard or tab, configurable parameters
  • Smart detection: if a child theme other than amar-design is active, show details without allowing reset
  • style.css editor: modify content from the backend with instant save
  • functions.php viewer: read-only textarea for quick inspection
  • Diagnostics: paths, permissions, file owner, write method (WP_Filesystem or direct)
  • Notifications: OPcache warning if changes aren't immediately visible, activation warning, auto-create notice
  • Divi Header Search: enable AJAX fullscreen overlay (if parent theme is Divi)

3.8 📝 Credits

  • Author name and website
  • Client name and website
  • Development notes (backend only)
  • Creation date

3.9 🛠 Tools

Dedicated tab for plugin detection and site tools checklist:

  • .htaccess — read-only viewer (if file exists)
  • Server-Side Cache — detection of OPcache, Memcache, Redis, APCu, persistent Object Cache, Reverse Proxy/Varnish, WP_CACHE constant
  • Client/Theme Cache — detection of Divi et-cache (file count if Divi active), 12 cache plugins (W3 Total Cache, WP Super Cache, WP Rocket, Flying Press, etc.), confirmation checkbox
  • SEO Plugin — detection of 11 SEO plugins (Yoast, Rank Math, SEOPress, AIOSEO, etc.)
  • Sitemap — detection of sitemap plugins (Google XML Sitemaps, built-in from SEO plugins)
  • Google Analytics & Search Console — detection of analytics plugins (Site Kit, MonsterInsights, ExactMetrics, etc.)
  • Backup — detection of backup plugins (UpdraftPlus, BackWPup, Duplicator, Jetpack, etc.) + hosting/off-site checklist
  • Security — detection of 10 security plugins (Wordfence, Solid Security, Sucuri, All In One WP Security, WPScan, Cerber, Defender, Anti-Malware, SecuPress, Jetpack Protect) + 5-item checklist (firewall, login protection, malware scan, SSL, auto updates)
  • Confirmation checkboxes for each section — mark what's configured and working

3.10 ⚙️ System

Informational tab with server diagnostics, SAF options, and self-test:

  • Server Info — PHP version, WordPress version, active theme, permalink structure, memory limit, max execution time, upload max
  • SAF Constants — list of all constants defined by SAF with their values
  • SAF Files — verify existence of all plugin files (admin.php, security.php, seo.php, etc.)
  • Language Files — verify loading of it_IT and en_US locales and translated key counts
  • SAF Options — list of saved database options with values
  • Full Self-Test — button to run automatic diagnostics verifying: mandatory SAF constants (12), plugin files (10), language files (2), translation keys (12 mandatory), DB options (3+), saf_write_file() function, WordPress constants (WP_DEBUG, WP_DEBUG_DISPLAY, DISALLOW_FILE_EDIT, WP_CACHE, WP_MEMORY_LIMIT)
  • Results with traffic light: ✅ OK, ⚠️ WARN, ❌ ERR

  • Branded login — custom logo and styling
  • Security headers — X-Content-Type-Options, X-Frame-Options

4.2 ⚙️ Admin (admin.php)

  • Settings registration and sanitizers
  • saf_get_org_data() helper — organization data array
  • Media uploader for logo
  • Branded orange admin menu
  • SMTP from name/email
  • Dashboard widget cleanup
  • Options cleanup on uninstall

4.3 🔍 SEO (seo.php)

  • JSON-LD Organization — homepage
  • JSON-LD WebSite — homepage
  • BreadcrumbList — JSON-LD breadcrumbs
  • Canonical URL — for all pages
  • Open Graph tags — og:title, og:description, og:image, og:url

Does not replace Rank Math / Yoast: these tags activate only when Rank Math is absent.

4.5 🚀 Performance (performance.php)

  • oEmbed disabled — blocks unnecessary external embeds

  • Lazy load — native images and iframes
  • CF7 optimized — load JS/CSS only where needed
  • jQuery migrate off — if no plugin requires it

4.6 🧹 Cleanup (cleanup.php)

  • Disable comments — when activated from Advanced tab
  • Hide admin menu — when configured
  • Admin bar — customized menu
  • Admin footer — branded with developer credits

4.7 📊 Dashboard (dashboard.php)

  • Pre-go-live checklist with colored progress bar
  • Site, theme, organization info
  • Quick buttons for Site Data and Guide

4.8 📖 Guide (guida.php)

Built-in documentation with 7 tabs:

  • Info — credits, license, disclaimer, guides
  • Structure — file map
  • Shortcodes — copy-paste reference
  • Security — active measures details
  • Checklist — pre-go-live with saved state

4.9 🖼 SVG Upload

Built into saf-loader.php for secure SVG file handling:

  • Enable from ⚙️ Site Data → Advanced → SVG Upload
  • DOM-based sanitization: whitelist of safe SVG tags and attributes
  • Automatic removal of scripts, event handlers (onclick, onload, onerror...) and javascript: href
  • Correct MIME support for .svg and .svgz files
  • WordPress filters: upload_mimes, wp_check_filetype_and_ext, wp_handle_upload_prefilter
⚠️ For maximum security in production, install the enshrined/svg-sanitize library via Composer.

5. Child Theme

The amar-design child theme is a preconfigured starter child theme provided by SAF. It is not created automatically — SAF asks the user whether to create it via a dashboard notice.

Why a child theme? If you modify parent theme files (Divi, Astra, Twenty Twenty-Four…), your changes get overwritten on every theme update. A child theme keeps your modifications safe.

5.1 Creating

  1. Go to ⚙️ Site Data → Child Theme
  2. If it doesn't exist, click "Create child theme amar-design"
  3. SAF copies style.css, screenshot.png, functions.php and inc/, css/, js/ folders
  4. Set the Template field to your parent theme folder (e.g. Divi, astra, twentytwentyfour)
  5. Activate from Appearance → Themes

If the child theme already exists, use the "Reset and recreate" button with double confirmation to delete everything and start fresh.

5.2 style.css

The style.css file has a required line:

Template: YOUR-PARENT-THEME-FOLDER

Replace YOUR-THEME with your parent theme's folder name (e.g. Divi, astra, twentytwentyfour).

Edit from backend: ⚙️ Site Data → Child Theme.

5.3 Divi Header Search

Enable from ⚙️ Site Data → Child Theme → Divi Features.

  • Adds magnifier icon in mobile menu
  • Fullscreen search overlay
  • Real-time AJAX search on post titles
  • Keyboard shortcut: Ctrl+K or ⌘K

7. Credits & Customization

Two credit levels:

Credits appear in:

  • Dashboard widget
  • WordPress admin footer
  • ⚙️ Site Data page

8. Uninstall

  1. Deactivate the plugin or remove saf-loader.php (if MU)
  2. Go to ⚙️ Site Data → Advanced → Delete all SAF options
  3. Or run SQL: DELETE FROM wp_options WHERE option_name LIKE 'saf_%';
Note: Saved options remain in the database even with the plugin deactivated. Use the cleanup button to remove them.
